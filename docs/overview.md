# 全体像

`mnu` は、x86_64 向けの `no_std` カーネルです。

このカーネルがやることは、次の 4 つに集約できます。

- ハードウェアを初期化する
- メモリを安全に分ける
- タスクを切り替える
- ユーザー空間の要求を syscall で受ける

## ひと目でわかる流れ

`bootloader -> entry.rs -> kernel_entry -> init -> kernel_main -> core.service`

起動後は、タイマー割り込みと syscall を使って、スケジューラが動きます。

## このカーネルの考え方

このカーネルは、便利さよりも「境界を明確にすること」を重視しています。

- ユーザー空間のポインタは信用しない
- 権限は privilege だけでなく capability で細かく分ける
- メモリ管理は物理と仮想を分離する
- 起動時にサービスの土台を作ってから、その上に機能を積む

## まず押さえる用語

- `Core`: カーネルモードで動く中核タスク
- `Service`: Ring 3 で動くが、システムサービスやドライバを担うタスク
- `User`: 一般アプリケーション
- `BootInfo`: ブートローダーが渡す起動情報
- `cext`: カーネルモジュール形式
