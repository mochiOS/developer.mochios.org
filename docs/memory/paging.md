# ページング

ページングは、仮想アドレスと物理アドレスの対応を管理します。

## このカーネルでの役割

- ブートローダーのページテーブルを引き継ぐ
- 自前の L4 テーブルを作る
- ユーザー空間が本当にマップされているか確認する
- 必要に応じてページを map/unmap する

## 重要な点

ユーザー空間のポインタは、値だけでは不十分です。

たとえば `0x1234` が見た目上はアドレスでも、現在のページテーブルで読めなければ意味がありません。

そのため、このカーネルではページテーブルを見て実際に参照できるかまで確認します。

## よく使う関数

- `init()`
- `init_page_table()`
- `map_page()`
- `switch_page_table()`
- `validate_user_ptr()` の土台になる range check

## 開発者向け

`syscall` や `exec` を触るときは、ページテーブルの切り替えとユーザーコピーの経路を必ず一緒に確認してください。
